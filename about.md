안녕하세요. 프론트엔드와 자동화, 배포 경험을 함께 다루는 개발자입니다.

서비스를 볼 때는 항상 두 가지를 같이 봅니다.

1. 사용자가 지금 이 화면을 얼마나 쉽게 이해할 수 있는가
2. 팀이 이 코드를 얼마나 오래 편하게 유지할 수 있는가

그래서 인터페이스를 만들 때도 예쁜 화면 하나보다 흐름과 문맥을 먼저 설계합니다. 배포 파이프라인을 만들 때도 멋진 도구보다 복구가 쉬운 구조를 우선합니다.

## 지금 집중하는 것

- 단순하고 빠른 배포 흐름 만들기
- 컴포넌트 경계를 명확하게 나누기
- 글을 통해 시행착오를 팀 지식으로 남기기

## 작업 방식

작은 변경을 자주 배포하고, 실제 사용성 피드백을 보고 다음 결정을 내리는 편입니다. 설계는 크게 보되 구현은 작게 나누는 것을 선호합니다.

## 다루는 주제

- 웹 프론트엔드 구조화
- Jenkins와 GitHub Pages 기반 배포
- 자동화 스크립트와 작업 흐름 개선
