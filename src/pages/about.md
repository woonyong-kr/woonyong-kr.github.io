프론트엔드와 배포 자동화를 중심으로, 작게 만들고 빠르게 검증하는 개발 과정을
기록합니다.

이 블로그는 [Astro](https://astro.build/) 기반의
[AstroPaper](https://github.com/satnaing/astro-paper) 테마를 바탕으로 구성했고,
Jenkins가 빌드한 결과물을 GitHub Pages에 배포하도록 맞췄습니다.

## 다루는 주제

- Astro와 React 같은 프론트엔드 구현 메모
- Jenkins, GitHub Pages, CI/CD 자동화 기록
- 작업 방식, 컴포넌트 설계, 제품 개발 회고

## 이 블로그의 방향

문장을 멋있게 포장하기보다, 실제로 배운 내용을 다시 써먹을 수 있게 남기는 쪽을
좋아합니다. 배포 설정, 코드 구조, 시행착오처럼 나중에 다시 찾게 되는 내용이
주된 기록 대상입니다.

## 운영 메모

- 정적 사이트 생성: Astro
- 테마 기반: AstroPaper
- 검색: Pagefind
- 배포: Jenkins -> `gh-pages` 브랜치 -> GitHub Pages

깔끔한 읽기 경험과 유지보수 가능한 배포 구성을 같이 챙기는 블로그를 목표로 두고
있습니다.
